# 跑道特征检测算法设计方案

> 2026-05-18 | 目标：RTX 3090 单帧 <60ms

---

## 1. 系统概述

检测飞机进近、着陆拉平、滑跑全过程中的跑道关键特征：4个角点、左右边线、中线。融合多源先验信息（GPS/IMU PnP投影、时序历史、跑道已知几何），输出逐像素 aleatoric 不确定性图，以及跑道相对于相机的6DoF位姿。通过 SE(3) 上的扩展卡尔曼滤波器进行时序平滑。

### 系统架构图

```
                          ┌──────────────────────────────────────┐
                          │       时序先验反馈（闭环）              │
                          │  EKF预测状态 → prev_prior + prev_lines │
                          └──────────────────────────────────────┘
                                          │
                                          v
 视频帧 ───────> [1. 特征网络] ──> 多通道热力图
 (3×256×256)      共享编码器        │  角点8ch + 边线4ch + 中线2ch
                  多头解码器        │  + 逐像素方差图
                    │  │  │
 PnP+几何先验 ─────^  │  │
 (4+3=7ch)            │  │
                      │  │
 时序先验 ────────────^  │
 (prev_prior: 4ch)      │
                         │
 前一帧线先验 ───────────^ 
 (prev_lines: 3ch)       
                         v
                  [2. 几何推理层]
                         │ DSNT提取角点
                         │ 加权PCA拟合直线
                         │ 计算消影点
                         │ 4角点PnP → 6DoF位姿（远场）
                         │ 2角点+VP PnP → 位姿（近场）
                         v
                  [3. SE(3)扩展卡尔曼滤波]
                         │ 过程模型：GPS/IMU
                         │ 观测模型：视觉检测
                         │ 预测状态反馈至下一帧输入
                         v
                  [4. 模式管理器]
                         │ 远场模式（4角点可见）/ 近场模式（2角点可见）
                         │ 基于GPS高度 + 角点可见性自动切换
```

**输入**：17通道张量 — RGB(3) + PnP角点热力图(4) + 几何先验(3：左边缘、右边缘、中线软分割图) + 时序先验(4：前一帧EKF预测角点热力图) + 前一帧线先验(3：前一帧检测边线/中线渲染图)。  
**输出**：4角点坐标（含不确定性）、3条直线参数（含不确定性）、消影点、6DoF位姿（含协方差）。  
**延时预算**：编码+解码≈16ms，几何求解≈3ms，EKF≈1ms，前后处理≈5ms → 总计约25ms，远低于60ms限制。

### 时序先验的闭环反馈

方案B的EKF在当前设计中仅做后处理平滑，**前一帧的检测结果没有作为先验反馈回特征网络**。此处新增两路时序先验，形成闭环：

1. **prev_prior（4通道）**：EKF预测步骤输出的上一帧后验角点坐标，渲染为4张高斯热力图（与PnP角点热力图相同的渲染方式，σ由EKF后验协方差对角线决定）。网络看到σ大的热力图时自动降低对该先验的信任。

2. **prev_lines（3通道）**：前一帧几何推理层输出的3条直线参数（左边缘、右边缘、中线），渲染为3张高斯带状软分割图（带宽σ≈8px），显式告诉网络上一帧检测到的线位置。

**闭环流程**：
```
Frame t-1: 网络检测 → EKF更新 → EKF预测t时刻状态
Frame t:   EKF预测状态 → 渲染prev_prior(4ch) + prev_lines(3ch) → 网络输入 → 网络检测 → ...
```

这样网络在每个时刻t都拥有三类先验的完整视图：
- **GPS/IMU先验**（7ch）：绝对地理参考，不依赖视觉历史，但精度受GPS误差影响
- **时序先验**（7ch）：基于视觉历史+运动模型预测，精度高但可能漂移
- 网络自主学会根据场景（光照变化、遮挡、GPS信号质量等）在两类先验间动态加权

## 2. 特征网络 — 共享编码器 + 多头解码器

### 2.1 网络结构

| 组件 | 详情 |
|------|------|
| 编码器 | HRNet-w18（预训练）或 UNet-ResNet18 — 复用现有架构 |
| 角点头 | `ConvBNSiLU` × 2 → `Conv2d(out=8)`。4个热力图通道（spatial softmax归一化）+ 4个方差通道（softplus激活）。对热力图做DSNT得到4个 `(x, y)` 归一化坐标（`[-1,1]`）。 |
| 边线头 | `ConvBNSiLU` × 2 → `Conv2d(out=4)`。2个sigmoid热力图（左边缘、右边缘）+ 2个方差通道。 |
| 中线头 | `ConvBNSiLU` × 2 → `Conv2d(out=2)`。1个sigmoid热力图 + 1个方差通道。 |

每个新增头约0.5M参数。模型总参数量约12-14M（基于HRNet-w18）。

### 2.2 不确定性建模 — Aleatoric + Epistemic

**Aleatoric 不确定性（数据噪声）**：通过异方差负对数似然（NLL）损失训练。

```
L_hm = Σ_p [ log(σ²_p) + (y_p - μ_p)² / σ²_p ]
```

角点头在DSNT坐标层面施加NLL：
```
L_coord = Σ_k [ log(σ²_k) + (x_gt,k - x_pred,k)² / σ²_k ]
```

其中 `μ_p` 为预测值，`σ²_p` 为每个像素/坐标的预测方差。σ² 大的区域自动降低该像素的loss权重——网络学会在难以预测的区域输出大方差。

**Epistemic 不确定性（模型不确定性）**：推理时执行 M=5 次 MC Dropout 前向传播，计算预测方差跨次传播的方差。这捕获模型容量不足或分布外输入带来的不确定性。

### 2.3 损失函数

总损失由6项组成：

```
L_total = λ₁·L_heatmap + λ₂·L_coord_nll + λ₃·L_edge + λ₄·L_centerline + λ₅·L_geom + λ₆·L_temporal
```

| 损失项 | 权重 | 公式 | 说明 |
|--------|------|------|------|
| `L_heatmap` | 1.0 | `MSE(H_pred, H_gt)` | 角点热力图逐像素MSE。GT为以标注角点为中心的2D高斯（σ≈2px）。仅对可见角点计算，不可见角点对应通道mask为0。驱动网络学会在角点位置产生高响应。 |
| `L_coord_nll` | 0.1 | `Σ_k [log(σ²_k) + (x_gt,k − μ_k)² / σ²_k]` | DSNT坐标的负对数似然损失。μ_k由DSNT从热力图提取，σ²_k由方差头预测。当预测偏差大时σ自动增大以降低loss贡献——网络学会在不确定区域输出大方差。 |
| `L_edge` | 0.5 | `Σ_p [log(σ²_p) + (H_edge(p) − H_gt(p))² / σ²_p]` | 边线（左边缘、右边缘）热力图的异方差NLL。GT由相邻角点连线渲染为高斯带状（σ≈8px带宽）。网络输出2个热力图通道+2个方差通道。 |
| `L_centerline` | 0.5 | `Σ_p [log(σ²_p) + (H_cl(p) − H_gt(p))² / σ²_p]` | 中线热力图的异方差NLL。GT = 左右边线的中位线，从4角点解析推导（bot_mid=(BL+BR)/2, top_mid=(TL+TR)/2），**零额外标注成本**。网络输出1个热力图通道+1个方差通道。 |
| `L_geom` | 0.05 | 见下方展开 | 几何一致性损失——软约束，确保角点、直线、消影点、位姿之间的几何自洽。权重较低以避免训练不稳定，但长期驱动网络向几何一致的解收敛。 |
| `L_temporal` | 0.2 | 见下方展开 | **新增**时序平滑损失——约束当前帧检测与前一帧EKF预测的一致性，抑制帧间抖动。 |

**L_geom 展开（4项）：**

```
L_geom = L_vp + L_corner_line + L_reproj + L_parallel
```

| 子项 | 公式 | 说明 |
|------|------|------|
| `L_vp` | `‖(l_left × l_right) − (l_left × l_center)‖₁` | 消影点一致性：左右边线交点应等于左边线与中线的交点 |
| `L_corner_line` | `Σ_k ‖ĉ_k − intersect(l_k1, l_k2)‖₂²` | 角线一致性：每个角点应在两条相邻线的交点附近 |
| `L_reproj` | `Σ_i ‖π(K(R·X_i+t)) − ĉ_i‖²` | 重投影误差：PnP位姿投影应与检测角点一致 |
| `L_parallel` | `1 − |cos(dir_entry, dir_exit)|` | 入口/出口边线平行约束（由需求第6条先验） |

**L_temporal 展开（3项）：**

```
L_temporal = L_corner_smooth + L_line_smooth + L_vp_smooth
```

| 子项 | 公式 | 说明 |
|------|------|------|
| `L_corner_smooth` | `Σ_k ‖ĉ_k(t) − ĉ_ekf,k(t)‖² / σ²_ekf,k` | 当前角点检测与EKF预测位置的一致性。σ²_ekf为EKF预测协方差——预测不确定性大时自动降低约束力度 |
| `L_line_smooth` | `Σ_m angle_dist(l_m(t), l_ekf,m(t))` | 当前线检测与EKF预测线方向的一致性（角度距离） |
| `L_vp_smooth` | `‖vp(t) − vp_ekf(t)‖₂` | 当前消影点与EKF预测消影点的一致性 |

**设计要点：**
- L_temporal在训练初期（前10 epoch）权重设为0，让网络先学会单帧检测，之后再逐渐引入时序约束
- L_temporal使用EKF预测值而非原始上一帧检测值作为target——EKF预测已经融合了IMU运动模型，比原始检测更平滑
- 训练时EKF以"teacher forcing"模式运行（使用GT初始化状态），推理时以自回归模式运行

边线和中线的伪标注通过以下方式生成：将相邻GT角点之间的连线渲染为高斯带状（σ≈8px带宽），沿整条线段均匀采样权重。

## 3. 几何推理层 — 可微后处理

所有步骤均在GPU上批量执行，支持端到端训练。

### 3.1 角点提取（DSNT）

沿用现有DSNT模块：对每个角点热力图做温度softmax → 计算空间网格上的期望坐标 `(x, y)`。
输出：`P_corners ∈ [-1,1]^(4×2)`，方差 `Σ_corners ∈ R^4`。

### 3.2 直线拟合（加权PCA）

给定一张热力图 `H ∈ [0,1]^(256×256)`（边线或中线）：

1. 构建权重图 `W[i,j] = H[i,j]`（mask掉 `H<0.1` 的像素）
2. 加权质心：`μ = Σ w_ij · [i,j] / Σ w_ij`
3. 加权协方差：`C = Σ w_ij · ([i,j]-μ)([i,j]-μ)^T / Σ w_ij`
4. 对2×2矩阵 `C` 做特征分解：较大特征值 λ₁ 给出沿线方向 `v`，λ₂ 给出垂直线散布
5. 齐次直线表示：`l = [a, b, c]` 使得 `ax+by+c=0`，其中 `[a,b]⊥v`，`c=−[a,b]·μ`

每线输出：`(l, σ²_line)`，其中 `σ²_line = λ₂`（垂直线散布即拟合不确定性）。

### 3.3 消影点计算

跑道左右边线在3D中平行 → 在图像上相交于消影点：
```
V_edges = l_left × l_right    （齐次坐标叉积）
```
中线也应经过同一消影点 → 用于一致性检验。

不确定性通过一阶误差传播从直线协方差矩阵传播到消影点。

### 3.4 PnP → 6DoF 位姿求解

**已知量：** 跑道宽度=60m，跑道长度≈3800m（北京首都机场36L），相机内参 *K*，跑道4个角点在ECEF坐标系下的3D坐标（通过GPS+跑道信息预计算）。

**远场PnP（4角点，进近阶段）：**

飞机在跑道入口前，4个角点全部可见。使用标准的4点PnP求解：

```
残差 = Σ_{i=1}^{4} ‖π(K(R·X_i + t)) − ĉ_i‖²_{Σ_i}
```

其中 ĉ_i 为DSNT提取的第i个角点像素坐标，Σ_i 为对应的协方差矩阵（由网络aleatoric方差给出）。4个3D-2D对应点提供足够约束求解6DoF位姿（EPnP闭式解初始化 + LM精化，3-5次迭代）。

**近场PnP（2角点 + 几何约束，跑道滑跑阶段）：**

飞机在跑道上，仅远端2个角点（far_left, far_right）可见。4点PnP退化为欠定问题，需引入额外几何约束：

| 残差项 | 来源 | 权重 |
|--------|------|------|
| 角点重投影（2个远端角点） | `‖π(K(RX_i+t)) − det_i‖²_Σi` | 1.0 |
| 消影点方向约束 | `‖VP_pred − VP(R,t)‖²` | 0.3 |
| 直线重投影（左右边线+中线） | `‖l_pred − l(R,t)‖²` | 0.2 |
| 跑道平面约束 | 位姿t的垂直分量 ≈ 飞机高度AGL | 0.1 |

其中 `π` 为透视投影，`(R,t)` 使用 se(3) 李代数参数化。消影点方向约束等价于约束相机光轴与跑道纵向的夹角，结合已知跑道宽度（60m）可唯一确定位姿。

**实现：** 使用 `torch.optim.LBFGS` 对残差向量优化，每帧5-10次迭代，批量并行。远场（4角点）初始化用EPnP闭式解；近场（2角点）初始化用上一帧EKF预测位姿。

### 3.5 模式相关行为（远场/近场切换）

术语统一：**远场** = 飞机在跑道入口前（进近阶段），4角点全部可见；**近场** = 飞机在跑道上（滑跑阶段），仅远端2角点可见。

| 模式 | 触发条件 | 可见角点 | 使用的线 | PnP输入 |
|------|---------|----------|---------|---------|
| 远场 | GPS高度 > 30m AGL 且 4角点热力图峰值均 > τ | 4个（全部） | 2边线 + 中线 | 4角点EPnP |
| 近场 | GPS高度 < 30m AGL 或 底部2角点峰值 < τ 连续5帧 | 2个（仅far_left, far_right） | 2边线 + 中线 | 2远端角点 + 消影点 + 线 + 高度先验 |

模式切换带滞回（±3帧去抖），避免在阈值边界抖动。近→远切回需同时满足高度 > 35m 且 底部角点峰值恢复 > τ。过渡期间对底部角点观测应用sigmoid权重渐变（5帧过渡），避免位姿跳变。

## 4. 时序滤波器 — SE(3) 扩展卡尔曼滤波

EKF承担两个角色：1）对网络检测结果做时序平滑，输出低抖动的位姿和特征；2）**将预测状态反馈为下一帧网络的时序先验输入**，形成闭环。

### 4.1 状态定义

`x = [t, ω]^T ∈ R^6`，其中 `t∈R³`（ENU坐标系下飞机相对于跑道原点的位置），`ω∈R³`（so(3)表示旋转）。

### 4.2 过程模型（GPS/IMU @ 30Hz）

```
x_k = f(x_{k-1}, u_k) + ε_k,   ε_k ~ N(0, Q_k)
```

`u_k` = IMU增量（加速度计+陀螺仪帧间积分）。  
`Q_k` 由IMU噪声参数推导（陀螺白噪声 + 加速度计噪声）。

当视觉帧率高于IMU时，仅用过程模型传播EKF；新视觉观测到达时再融合更新。

### 4.3 观测模型（视觉 @ 视频帧率）

```
z_k = h(x_k) + η_k,   η_k ~ N(0, R_k)
```

观测向量 `z`：
- 远场模式：`[角点(4×2), 边线(2×3), 中线(1×3)]` = 17维
- 近场模式：`[远端角点(2×2), 边线(2×3), 中线(1×3)]` = 13维

`R_k` 为**块对角**协方差矩阵，由网络aleatoric不确定性 + 几何求解器误差传播拼接而成。

`h(·)` 将6DoF状态通过 *K* 投影到图像坐标 / 直线参数。

### 4.4 时序先验反馈（关键新增）

EKF在每个帧周期执行两步操作后，将预测状态渲染为先验图反馈给下一帧网络输入：

```
Frame t-1 完成后:
  1. EKF update: 融合t-1时刻视觉观测，得到后验状态 x_{t-1|t-1}, P_{t-1|t-1}
  2. EKF predict: 用IMU预测量预测t时刻状态 x_{t|t-1}, P_{t|t-1}
  3. 渲染prev_prior: 将 x_{t|t-1} 中的位姿投影到图像 → 4个预测角点位置
     → 渲染4通道高斯热力图，每通道σ²由 P_{t|t-1} 的角点位置协方差对角线决定
  4. 渲染prev_lines: 将 x_{t|t-1} 投影得到3条预测线 → 渲染3通道高斯带状图

Frame t 处理时:
  5. 网络接收 prev_prior(4ch) + prev_lines(3ch) 作为额外输入
  6. 网络检测 → EKF update → ...（循环）
```

**自适应先验σ：** 当EKF预测协方差大（如GPS信号差、IMU漂移大），prev_prior热力图的σ自动变大——网络看到"模糊"的先验，自动降低对其的信任。协方差闭环是系统优雅降级的核心机制。

### 4.5 可选：RTS平滑器

如果允许2-3帧延迟，可运行 Rauch–Tung–Striebel 后向平滑，获得更优的历史轨迹估计。默认关闭。

## 5. 训练策略（4个阶段）

### 阶段1 — 角点预训练（复用现有checkpoint）
- 加载现有角点模型（`pnp_noise_std=5.0` 训练）
- 冻结编码器，添加方差头，仅训练角点NLL → 约10 epochs

### 阶段2 — 加入边线和中线头
- 冻结编码器，训练Edge + Centerline头，使用NLL热力图损失
- 伪标注：由GT角点连线渲染为高斯带状
- 约20 epochs

### 阶段3 — 端到端 + 几何损失
- 解冻全部网络
- 加入几何一致性损失（消影点、直线共面性、PnP重投影）
- 使用低权重（相对热力图损失的 0.05×）避免训练不稳定
- 约30 epochs

### 阶段4 — EKF调优（可选）
- 固定网络权重，在视频序列上运行
- 通过验证集轨迹的MLE优化过程噪声 `Q` 和初始状态不确定度 `P₀`

### 数据增强

| 增强项 | 范围 |
|--------|------|
| PnP噪声标准差 | 3-8 px（模拟GPS/IMU误差） |
| Crop随机抖动 | half-size的±5%（模拟不完美PnP裁剪） |
| 亮度 | 0.7-1.3× |
| 对比度 | 0.8-1.2× |

---

## 6. 逐帧推理流水线

```
1. 接收当前帧 + 对应时间戳的GPS/IMU姿态
2. 根据GPS位置判断模式（全场/远场）
3. 渲染PnP角点热力图(4ch) + 几何先验(3ch)
4. 根据PnP计算裁剪区域 → 裁剪并缩放至256×256
5. 网络前向传播：编码 → 3头解码 → 热力图 + 方差（≈15ms）
6. 几何求解：DSNT + 加权PCA + 消影点 + LM PnP（≈3ms）
7. 视觉观测+协方差 喂入EKF → 平滑6DoF + 协方差（≈1ms）
8. 输出：
   - 远场模式：远端2角点 + 3条直线 + 6DoF
   - 全场模式：4角点 + 3条直线 + 6DoF
   - 始终输出：每个输出的不确定性 + EKF协方差矩阵
```

## 7. 延时预算

| 步骤 | 目标 (ms) |
|------|-----------|
| 预处理（渲染先验、裁剪、转tensor） | 2 |
| 编码器前向 | 8 |

余量充足：不到60ms预算的1/3，后续扩展空间大。

## 8. 关键设计决策及其理由

- **热力图而非直接回归**做边线/中线：空间归纳偏置利于泛化；PCA对热力图加权可自然处理长条状模糊特征。
- **多头解码而非多模型**：单次前向传播，共享特征，无调度开销。
- **EKF而非因子图优化**：每帧计算量确定（无优化时间的不确定性）；Q和R可在线调优。
- **GPS驱动模式切换而非端到端学习**：入口附近GPS可靠性高（米级）；避免网络隐式学习模式检测。
- **从现有GT角点推导伪标注**：零新增标注成本；对于平坦跑道，边线/中线标注在几何上是精确的。

## 9. 待扩展问题

1. **夜间/低能见度运行** — 当前设计假设日间视觉条件；网络可能需要红外或低光增强。
2. **多跑道场景** — 若画面中出现两条跑道，需要上游跑道分配模块。
3. **相机内参漂移** — 如果相机在云台上或具有可变焦距，内参需在运行时更新。
4. **传感器信号丢失** — GPS/IMU丢失时的处理（纯视觉模式，高不确定性）。
